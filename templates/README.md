# Templates

Reusable templates for common development tasks and workflows.

## What are Templates?

Templates provide standardized starting points for:
- Project structures
- Configuration files
- Documentation
- Common code patterns
- Workflow definitions

## Categories

### Project Templates
- Starter projects for different frameworks
- Boilerplate configurations
- Standard directory structures

### Documentation Templates
- README templates
- API documentation
- Architecture decision records (ADRs)
- CLAUDE.md templates

### Code Templates
- Common patterns and implementations
- Test suites
- CI/CD configurations

### Workflow Templates
- Pull request templates
- Issue templates
- Planning documents

## Using Templates

1. Copy the template to your target location
2. Replace placeholder values
3. Customize for your specific needs
4. Remove unused sections

## Creating New Templates

1. Identify commonly repeated patterns
2. Extract the reusable structure
3. Mark customizable sections with placeholders
4. Document required customizations
5. Include usage examples

## Placeholders

Use these common placeholders:
- `{{PROJECT_NAME}}`: Project name
- `{{AUTHOR}}`: Author name
- `{{DATE}}`: Current date
- `{{DESCRIPTION}}`: Project description
- `{{VERSION}}`: Version number
